# 5. PRD 초안 요청
> 프로젝트 요구사항 문서(PRD)를 작성하기 위한 프롬프트

---

## 📄 기본 PRD 요청

```
"내가 만들 프로젝트의 PRD 초안을 작성해줘. 
다음에 첨부한 주요 구성 요소를 포함해서.

PRD 주요 구성 요소:
- 프로젝트 개요 (대상 사용자, 앱의 목표)
- 핵심 기능
- 기술 스택
- 향후 추가 기능
- 성공 지표"
```

---

## 🔄 PRD 수정 요청

```
"백엔드는 파이어베이스를 사용하는 걸로 해주고, 
커뮤니티 기능을 위주로 PRD를 다시 작성해줘."
```

---

## 📋 상세 PRD 템플릿 요청

```
"[프로젝트명]의 상세 PRD를 작성해줘:

1. 프로젝트 개요
   - 프로젝트 배경 및 목적
   - 대상 사용자 (페르소나 참조)
   - 해결하려는 문제

2. 핵심 기능 명세
   - 기능별 상세 설명
   - 우선순위 (P0, P1, P2)
   - 사용자 스토리

3. 기술 스택
   - 프론트엔드: [Next.js, React 등]
   - 백엔드: [Firebase, Supabase 등]
   - 데이터베이스: [Firestore, PostgreSQL 등]
   - 인프라: [배포 환경]

4. 향후 추가 기능 (Roadmap)
   - Phase 1: MVP
   - Phase 2: 확장 기능
   - Phase 3: 수익화

5. 성공 지표 (KPIs)
   - DAU/MAU 목표
   - 리텐션 목표
   - 수익 목표"
```

---

## 🎯 PRD 심화 질문

```
"이 PRD에서 기술적으로 가장 어려운 부분은 뭐야?"
"MVP 범위를 줄이려면 어떤 기능을 빼야 할까?"
"이 기능들 중 Firebase로 구현하기 어려운 것은?"
```
