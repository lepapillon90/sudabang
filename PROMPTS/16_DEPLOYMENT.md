# 16. 배포 및 운영
> 앱 배포와 운영을 위한 프롬프트

---

## 🚀 배포 설정 요청

```
"앱을 [Vercel/Firebase Hosting/AWS]에 배포하기 위한 설정을 해줘:

필요한 설정:
- 빌드 명령어
- 환경 변수 설정
- 도메인 연결
- HTTPS 설정

단계별로 안내해줘."
```

---

## 📦 빌드 최적화

```
"프로덕션 빌드를 최적화해줘:

체크할 항목:
- 번들 사이즈 분석
- Tree shaking
- 코드 스플리팅
- 이미지 최적화
- 캐싱 전략

최적화 전후 비교를 보여줘."
```

---

## 🔧 CI/CD 파이프라인

```
"GitHub Actions를 사용한 CI/CD 파이프라인을 만들어줘:

워크플로우:
1. PR 생성 시: 린트 + 테스트
2. main 브랜치 푸시 시: 빌드 + 배포

.github/workflows/deploy.yml 파일을 작성해줘."
```

---

## 📊 모니터링 설정

```
"앱 모니터링을 위한 설정을 해줘:

1. 에러 트래킹: Sentry
2. 성능 모니터링: Google Analytics / Vercel Analytics
3. 로그 관리: [선택]

각 서비스의 설정 방법을 알려줘."
```

---

## 🌐 환경별 설정

```
"개발/스테이징/프로덕션 환경을 분리해줘:

- Development: 로컬 개발용
- Staging: 테스트 배포용
- Production: 실제 서비스용

각 환경별 환경 변수와 설정 차이를 정리해줘."
```

---

## 🔒 보안 체크리스트

```
"배포 전 보안 체크리스트를 만들어줘:

- [ ] 환경 변수 노출 확인
- [ ] API 키 보호
- [ ] CORS 설정
- [ ] HTTPS 강제
- [ ] 인증 토큰 관리
- [ ] XSS/CSRF 방어
- [ ] 의존성 취약점 검사"
```
