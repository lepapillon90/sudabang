# 10. 와이어프레임 요청
> 앱 UI의 기본 구조를 설계하기 위한 프롬프트

---

## 🖼️ 전체 와이어프레임 요청

```
"우리 앱의 전체 와이어프레임을 설계해줘.
모든 주요 화면과 화면 간의 연결 관계를 보여줘."
```

**상세 버전:**
```
"[프로젝트명]의 전체 와이어프레임을 만들어줘:

1. 화면 목록
   - 온보딩/스플래시
   - 로그인/회원가입
   - 메인(홈)
   - 핵심 기능 화면들
   - 설정/프로필
   
2. 화면 간 네비게이션 흐름
3. 공통 컴포넌트 (탭바, 헤더 등)

각 화면의 목적과 주요 요소를 포함해줘."
```

---

## 🏠 홈 화면 와이어프레임

```
"홈 화면의 와이어프레임을 설계해줘.
사용자가 앱에 들어왔을 때 첫 번째로 보게 되는 화면이야.

포함할 요소:
- 헤더 (로고, 알림, 프로필)
- 메인 콘텐츠 영역
- 하단 네비게이션 바
- 주요 CTA 버튼"
```

---

## 📱 기능별 화면 와이어프레임

```
"[기능명] 화면의 와이어프레임을 설계해줘:

화면 목적: [사용자가 이 화면에서 할 일]

포함할 요소:
- 상단 영역: [헤더, 검색바 등]
- 중앙 영역: [메인 콘텐츠]
- 하단 영역: [버튼, 네비게이션 등]

각 요소의 위치와 대략적인 크기 비율도 알려줘."
```

---

## ✍️ 글쓰기/입력 화면 와이어프레임

```
"글쓰기(또는 입력) 화면의 와이어프레임을 설계해줘:

- 제목 입력 필드
- 본문 입력 영역
- 이미지/파일 첨부 버튼
- 카테고리/태그 선택
- 저장/게시 버튼
- 취소/뒤로가기"
```

---

## 👤 프로필 화면 와이어프레임

```
"프로필 화면의 와이어프레임을 설계해줘:

- 프로필 이미지 및 기본 정보
- 활동 통계 (게시글 수, 팔로워 등)
- 내 게시물/활동 목록
- 설정 및 편집 버튼
- 로그아웃/계정 관리"
```

---

## 📋 PRD 기반 와이어프레임

```
"PRD에 정의된 기능들을 바탕으로 와이어프레임을 만들어줘.

PRD의 각 핵심 기능에 대해:
1. 해당 기능이 위치할 화면
2. UI 요소 배치
3. 사용자 인터랙션 방식
을 시각적으로 설계해줘."
```

---

## 🔍 와이어프레임 심화 질문

```
"이 와이어프레임에서 UX를 개선할 수 있는 부분은?"
"사용자 동선이 가장 효율적인 레이아웃은?"
"접근성(Accessibility)을 고려한 수정 사항은?"
```

---

## 📐 ASCII 와이어프레임 요청

```
"[화면명]의 와이어프레임을 ASCII 아트로 그려줘.
각 요소의 위치와 크기를 텍스트로 표현해줘."
```

**예시 출력:**
```
┌─────────────────────────────┐
│  🏠 로고    🔔  👤          │ ← 헤더
├─────────────────────────────┤
│                             │
│    ┌─────────────────┐      │
│    │   메인 배너      │      │
│    └─────────────────┘      │
│                             │
│  ┌─────┐ ┌─────┐ ┌─────┐   │
│  │카드1│ │카드2│ │카드3│   │ ← 콘텐츠
│  └─────┘ └─────┘ └─────┘   │
│                             │
├─────────────────────────────┤
│  🏠    📝    👤    ⚙️      │ ← 탭바
└─────────────────────────────┘
```
