# 15. 테스트 및 품질 관리
> 앱 테스트와 품질 보장을 위한 프롬프트

---

## 🧪 테스트 코드 요청

```
"[컴포넌트/함수명]의 테스트 코드를 작성해줘:

테스트 케이스:
1. 정상 동작 테스트
2. 엣지 케이스 테스트
3. 에러 케이스 테스트

Jest/React Testing Library를 사용해줘."
```

---

## 📋 테스트 시나리오 작성

```
"[기능명]의 테스트 시나리오를 작성해줘:

QA 체크리스트:
- [ ] 정상 케이스: [설명]
- [ ] 경계값 테스트: [설명]
- [ ] 에러 처리: [설명]
- [ ] 권한 테스트: [설명]
- [ ] 성능 테스트: [설명]"
```

---

## 🔄 E2E 테스트

```
"[유저 플로우]의 E2E 테스트를 작성해줘:

사용자 시나리오:
1. 로그인
2. 메인 화면 진입
3. [기능] 사용
4. 결과 확인

Playwright/Cypress로 작성해줘."
```

---

## 🐛 디버깅 요청

```
"다음 에러를 디버깅해줘:

에러 메시지: [에러 내용]
발생 상황: [어떤 상황에서 발생했는지]
예상 동작: [어떻게 동작해야 하는지]
실제 동작: [어떻게 동작하는지]

원인 분석과 해결 방법을 알려줘."
```

---

## ⚡ 성능 최적화

```
"[컴포넌트/페이지]의 성능을 최적화해줘:

확인할 항목:
- 불필요한 리렌더링
- 메모이제이션 적용
- 코드 스플리팅
- 이미지 최적화
- 번들 사이즈

개선 전후 비교도 보여줘."
```

---

## 📱 크로스 브라우저/디바이스 테스트

```
"앱의 호환성 테스트 체크리스트를 만들어줘:

브라우저:
- [ ] Chrome
- [ ] Safari
- [ ] Firefox
- [ ] Edge

디바이스:
- [ ] iOS (iPhone, iPad)
- [ ] Android (다양한 화면 크기)
- [ ] Desktop"
```

---

## 🔍 코드 리뷰 요청

```
"다음 코드를 리뷰해줘:

[코드 블록]

체크할 항목:
- 코드 품질
- 성능 이슈
- 보안 취약점
- 베스트 프랙티스
- 개선 제안"
```
