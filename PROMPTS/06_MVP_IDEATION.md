# 6. MVP 아이디어 도출
> 최소 기능 제품(MVP)을 정의하기 위한 프롬프트

---

## 🎯 MVP 요청 프롬프트

```
"지금 진행된 것들로 앱을 만들고 싶어. 
이에 대해 MVP 단계의 앱 아이디어를 제안해줘."
```

---

## 📦 MVP 도출을 위한 심화 프롬프트

```
"MVP를 만든다면:
1. 반드시 있어야 할 핵심 기능 3개는?
2. 나중에 추가할 수 있는 기능은?
3. 2주 안에 만들 수 있는 범위는?
4. 사용자 검증을 위해 필요한 최소 기능은?"
```

---

## 🗺️ MVP 로드맵 요청

```
"MVP에서 풀 버전까지 로드맵을 만들어줘:
1. Phase 1 (MVP): 핵심 기능만
2. Phase 2: 사용자 피드백 반영
3. Phase 3: 수익화 기능
4. Phase 4: 확장 기능"
```

---

## 🔥 MVP 결정 팁

```
💡 아이디어 → MVP 전환 시 항상 물어볼 것:

"이 기능이 없으면 앱이 작동하지 않아?" 
→ NO라면 MVP에서 제외

"사용자가 이 기능 때문에 앱을 쓸까?"
→ YES라면 MVP에 포함
```
