# 13. 기능 구현 요청
> 실제 코드 구현을 요청하기 위한 프롬프트

---

## 🚀 기본 기능 화면 추가 요청

```
"[기능명] 화면을 구현해줘.

화면 요구사항:
- 화면 목적: [사용자가 이 화면에서 할 일]
- 필요한 UI 컴포넌트: [리스트, 카드, 버튼 등]
- 데이터 연동: [API 호출 또는 로컬 데이터]
- 상태 관리: [필요한 state]

디자인 시스템에 맞춰 구현해줘."
```

---

## 🃏 카드/위젯 컴포넌트 생성

```
"PostCard 위젯을 생성해줘:

Props:
- title: 제목
- content: 내용
- author: 작성자
- date: 날짜
- image?: 이미지 (선택)

기능:
- 클릭 시 상세 페이지로 이동
- 좋아요/북마크 버튼
- 공유 버튼

Tailwind CSS로 스타일링해줘."
```

---

## 📋 리스트/목록 컴포넌트

```
"ListView 컴포넌트를 만들어줘:

기능:
- 무한 스크롤 또는 페이지네이션
- 로딩 상태 표시
- 빈 상태 표시
- 에러 상태 표시
- Pull to Refresh (모바일)

PostCard를 아이템으로 사용해줘."
```

---

## 🔀 화면 라우팅 설정

```
"앱의 라우팅을 설정해줘:

라우트 구조:
/                 → 홈
/login            → 로그인
/signup           → 회원가입
/posts            → 게시글 목록
/posts/:id        → 게시글 상세
/posts/create     → 게시글 작성
/profile          → 내 프로필
/profile/:userId  → 다른 사용자 프로필
/settings         → 설정

인증이 필요한 라우트와 공개 라우트를 구분해줘."
```

---

## 🔐 인증 기능 구현

```
"인증(Authentication) 기능을 구현해줘:

1. 로그인 화면
   - 이메일/비밀번호 로그인
   - 소셜 로그인 (Google, Apple)

2. 회원가입 화면
   - 폼 유효성 검사
   - 이메일 인증

3. 로그아웃 기능

4. 인증 상태 관리
   - 로그인 상태 유지
   - Protected Route

Firebase Auth를 사용해줘."
```

---

## 📝 CRUD 기능 구현

```
"[리소스명]의 CRUD 기능을 구현해줘:

- Create: 새로운 [리소스] 생성 폼
- Read: [리소스] 목록 및 상세 조회
- Update: [리소스] 수정 폼
- Delete: [리소스] 삭제 (확인 모달)

Firestore를 사용하고, 에러 핸들링도 포함해줘."
```

---

## 🔔 알림/토스트 시스템

```
"앱 전역에서 사용할 토스트 알림 시스템을 만들어줘:

타입:
- success (성공)
- error (에러)
- warning (경고)
- info (정보)

기능:
- 자동 닫힘 (3초)
- 수동 닫기 버튼
- 여러 개 동시에 표시"
```

---

## 📱 반응형 레이아웃

```
"앱의 반응형 레이아웃을 구현해줘:

브레이크포인트:
- Mobile: ~640px
- Tablet: 641px~1024px
- Desktop: 1025px~

각 브레이크포인트에서 레이아웃이 어떻게 변하는지 정의해줘."
```
