# 3. 페르소나 초안 요청
> 타겟 사용자를 구체화하기 위한 프롬프트

---

## 👤 기본 페르소나 요청

```
"내가 만들 프로젝트의 핵심 사용자 페르소나를 2명 만들어줘. 
이름, 나이, 직업, 경험 수준, 앱 사용 동기, 불편한 점 등을 포함해서."
```

---

## 📋 상세 페르소나 요청 템플릿

```
"[프로젝트명]의 사용자 페르소나를 만들어줘:

포함할 정보:
- 기본 정보: 이름, 나이, 직업, 거주지
- 기술 수준: 앱/웹 사용 능숙도
- 목표: 이 앱으로 달성하고 싶은 것
- 동기: 왜 이 앱을 사용하게 되었는지
- 불편한 점: 기존 서비스에서 겪는 문제
- 행동 패턴: 앱을 언제, 어떻게 사용할지
- 기대 사항: 이 앱에서 원하는 것"
```

---

## 🎯 페르소나 활용 심화 질문

```
"이 페르소나가 우리 앱의 어떤 기능을 가장 많이 사용할까?"
"페르소나 A와 B의 사용 패턴 차이점은?"
"이 페르소나를 만족시키기 위한 핵심 기능은?"
```
